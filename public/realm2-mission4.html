1. The Basics of Cryptography in Bitcoin

Bitcoin relies on several cryptographic primitives that work together to create a secure, trustless system for transferring value without intermediaries. Let's explore the key components:

1

Hash Functions

A hash function takes input data of any size and produces a fixed-size output (the "hash"). Bitcoin uses the SHA-256 hash function, which generates a 256-bit (32-byte) hash.

Key properties of hash functions in Bitcoin:

  • One-way function: Easy to compute a hash from input data, but practically impossible to derive the original input from the hash.
  • Deterministic: The same input always produces the same hash output.
  • Avalanche effect: A small change in the input data causes a large, unpredictable change in the hash output.
  • Collision resistance: It's extremely difficult to find two different inputs that produce the same hash output.
Example: The hash of "Bitcoin" using SHA-256 is:
6b88c087247aa2f07ee1c5956b8e1a9f4c7f892a70e324f1bb3d161e05ca107b
2

Public-Private Key Pairs

Bitcoin uses asymmetric (or public-key) cryptography, which relies on a pair of mathematically related keys:

  • Private key: A randomly generated 256-bit number that must be kept secret.
  • Public key: Derived from the private key using elliptic curve multiplication, which can be safely shared.

The critical property is that while it's easy to derive a public key from a private key, it's computationally infeasible to reverse the process and derive the private key from a public key.

Important: Your private key is effectively your access to your bitcoin. Anyone with your private key can spend your bitcoin, and if you lose it, you lose access to your funds permanently.
3

Digital Signatures

Digital signatures in Bitcoin use the ECDSA (Elliptic Curve Digital Signature Algorithm) and provide:

  • Authentication: Proof that the transaction was created by the owner of the private key.
  • Non-repudiation: The signer cannot deny having signed the transaction.
  • Integrity: The transaction cannot be altered after being signed.

The signature process works as follows:

  1. The transaction data is hashed.
  2. The hash is signed using the sender's private key, creating a digital signature.
  3. Anyone can verify the signature using the sender's public key, confirming that the transaction was authorized by the owner of the private key.

2. Bitcoin Addresses

Bitcoin addresses are not directly stored on the blockchain. Instead, they are derived from public keys through a series of transformations:

  1. Start with a public key
  2. Apply a hash function (SHA-256, then RIPEMD-160)
  3. Add a version byte (to indicate the address type)
  4. Calculate a checksum and append it
  5. Encode the result in Base58Check encoding

This produces the familiar Bitcoin addresses beginning with '1', '3', or 'bc1':

3. The Role of Cryptography in Bitcoin Transactions

1

Transaction Inputs and Outputs

Bitcoin transactions consist of inputs (sources of bitcoin) and outputs (destinations):

  • Inputs: Reference previous transaction outputs that are being spent.
  • Outputs: Specify the recipient addresses and amount to send.

Each input must include a valid digital signature proving that the sender has the authority to spend those bitcoin.

2

Transaction Verification

Before being added to the blockchain, each transaction is verified by nodes in the network:

  1. Check that all inputs reference valid, unspent transaction outputs (UTXOs).
  2. Verify that the total input value is greater than or equal to the total output value.
  3. Verify the digital signature for each input, ensuring the sender has the right to spend those bitcoin.

This verification process happens across thousands of nodes, without requiring trust in any central authority.

3

The Blockchain and Merkle Trees

Bitcoin uses Merkle trees (hash trees) to efficiently verify that a transaction is included in a block:

  • All transactions in a block are hashed, then paired and hashed again recursively until a single hash remains (the Merkle root).
  • The Merkle root is included in the block header, which is then hashed during mining.
  • This structure allows for efficient verification that a transaction is included in a block without needing to download the entire blockchain.

4. Security Implications of Cryptography in Bitcoin

The cryptographic foundations of Bitcoin provide several important security properties:

  • Decentralized verification: Anyone can verify the validity of transactions and blocks without trusting a central authority.
  • Immutability: Once a transaction is confirmed and buried under subsequent blocks, it becomes practically impossible to alter.
  • Censorship resistance: No central authority can freeze accounts or prevent valid transactions from being processed.
  • Self-custody: Users can maintain direct control over their bitcoin without relying on third parties to hold their funds.
Key Insight: Bitcoin replaces trust in institutions with cryptographic verification. Instead of trusting banks or payment processors to maintain accurate ledgers and honor transactions, Bitcoin users can mathematically verify the entire system.
